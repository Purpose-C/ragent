# 阅读顺序与使用说明

## 这套笔记适合谁

这套笔记面向刚接触 Java 项目、Spring Boot 和 RAG 的学习者。阅读目标不是背类名，而是学会沿着“页面 -> 接口 -> Service -> 核心组件 -> 数据库/模型”的方向追踪真实调用链。

## 推荐阅读路线

| 阶段 | 推荐阅读文档 | 学习目标 | 验收标准 |
|---|---|---|---|
| 准备 | 00、01、14 | 建立术语和全局地图 | 能解释 RAG、Embedding、SSE |
| 跑通 | 02、03、04 | 理解环境、模块和启动过程 | 能找到两个启动类和主配置 |
| 数据入库 | 05、06 | 看懂表结构与入库 Pipeline | 能口述 Fetcher 到 Indexer |
| 智能问答 | 07、08、09 | 看懂 RAG、模型容错和 MCP | 能画出一次问答调用链 |
| 前后端联调 | 10 | 从页面反查接口 | 能从聊天页找到后端 Controller |
| 巩固 | 11、12、13、15 | Debug、改造、表达和 Git | 能完成项目讲解与改造方案 |

## 配合工具学习

1. IDE 使用 IntelliJ IDEA，先用 `Shift` 连按两次搜类名，再用“Find Usages”看调用方。
2. Debug 时一次只观察一条链路。先在 Controller 断点，再逐层 Step Into。
3. 普通接口可用 Postman/Apifox；SSE 建议用浏览器页面或 `curl -N`。
4. 数据库用 DBeaver/IDEA Database 查看 PostgreSQL 表，先只读查询。
5. 前端在浏览器 DevTools 的 Network 中查看 URL、参数、响应和 EventStream。

## 两周使用方法

每天 2~4 小时，约一半时间读文档，一半时间在 IDE 中定位真实代码。每天结束前完成三件事：画一张小图、口述一条调用链、写下一个仍不明白的问题。详细安排见 `11-两周学习计划.md`。

## 看不懂源码怎么办

采用“五步法”：先看类名职责，再看依赖字段，再看公开方法，再找调用方，最后才钻进私有实现。遇到泛型、异步或设计模式时，先用一句中文写出输入和输出，不要一开始纠结所有语法。

## 建议补齐的基础

- Java：接口、继承、集合、异常、Lambda、线程池、`CompletableFuture`。
- Spring Boot：依赖注入、Bean、Controller、Service、配置绑定、AOP。
- 数据库：SQL、索引、事务、JSONB、向量字段的基本概念。
- HTTP：GET/POST、JSON、状态码、SSE。
- 前端：TypeScript、React 组件、Hook、Axios、路由。
- AI：Prompt、Token、Embedding、余弦相似度。
