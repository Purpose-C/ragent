# 本地启动指南

## 环境清单

根 `pom.xml` 明确要求 Java 17。前端 `frontend/package.json` 使用 Vite，仓库包含 `package-lock.json`，因此优先使用 npm。Node 具体最低版本未在仓库明确声明，建议使用当前维护中的 Node LTS；实际版本需要运行验证。

| 步骤 | 操作 | 作用 | 相关文件 | 常见问题 |
|---|---|---|---|---|
| 1 | 克隆仓库 | 获取源码 | `README.md` | SSH Key 未配置 |
| 2 | 准备 JDK 17、Maven | 构建 Java | `pom.xml` | `JAVA_HOME` 错误 |
| 3 | 启动 PostgreSQL/Redis/RocketMQ/RustFS | 后端依赖 | `application.yaml`、`resources/docker` | 端口或密码不一致 |
| 4 | 初始化 PostgreSQL | 创建 21 张表与数据 | `resources/database/schema_pg.sql`、`init_data_pg.sql` | 未安装 vector 扩展 |
| 5 | 配置模型密钥 | 调用 AI | `application.yaml` | 环境变量为空 |

## 克隆与构建

```bash
git clone git@github.com:你的用户名/ragent.git
cd ragent
./mvnw clean package -DskipTests
```

也可使用系统 Maven。若依赖下载慢，检查 Maven 镜像配置，不要随意修改项目 POM。

## 数据库初始化

1. 创建名为 `ragent` 的 PostgreSQL 数据库。
2. 确保安装 pgvector 扩展。
3. 顺序执行 `resources/database/schema_pg.sql`、`resources/database/init_data_pg.sql`。
4. 老版本升级才使用 `upgrade_v1.0_to_v1.1.sql` 等脚本，新库不要重复执行升级脚本。

## 中间件

## 必改配置

- PostgreSQL：`spring.datasource.url/username/password`。
- Redis：`spring.data.redis.host/port/password`。
- RocketMQ：`rocketmq.name-server`。
- 对象存储：`rustfs.url/access-key-id/secret-access-key`。
- AI 密钥：环境变量 `BAILIAN_API_KEY`、`AIHUBMIX_API_KEY`、`SILICONFLOW_API_KEY`，按实际模型选择配置。

密钥、密码、本地 `.env` 和生产地址不要提交。示例只能写 `your-api-key`、`your-password`。

## 启动命令

```bash
./mvnw -pl mcp-server spring-boot:run
./mvnw -pl bootstrap spring-boot:run
cd frontend
npm install
npm run dev
```

前端通常为 Vite 的 `http://localhost:5173`；后端为 `http://localhost:9090/api/ragent`；MCP 为 9099。前端代理细节见 `frontend/vite.config.ts`。

## 如何判断成功

- 后端日志无数据库、Redis、RocketMQ连接异常。
- 登录页可打开，Network 请求指向 `/api/ragent`。
- 登录后可加载 `/user/me`、`/conversations`。
- 提问后 Network 中 `/rag/v3/chat` 为 EventStream 并持续收到事件。
- 管理后台能读到知识库、入库 Pipeline 和 Trace。

## 常见排查

| 现象 | 优先检查 |
|---|---|
| 数据库启动失败 | 数据库名、密码、vector 扩展、脚本是否执行 |
| 401/未登录 | Sa-Token、Authorization 头、初始化用户 |
| 问答立即失败 | 模型 API Key、候选模型、网络和维度配置 |
| 上传失败 | RustFS、文件大小限制、Pipeline 配置 |
| MCP 工具缺失 | 9099 服务、`rag.mcp.servers`、工具发现日志 |
| 向量报维度错误 | `rag.default.dimension` 与 Embedding 输出一致性 |
