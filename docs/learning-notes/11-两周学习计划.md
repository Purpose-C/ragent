# 两周学习计划

> 本计划把 Ragent 从“跑起来”到“能讲出来”拆成 14 天。每天 2～4 小时，约一半读文档，一半跟源码/动手。目标不是背类名，而是能在 IDE 里独立打断点、能在数据库里验证结果、能口述一条完整调用链。

---

## 使用说明

1. **按顺序执行**：Day 1～7 重跑通和入库，Day 8～14 重 RAG、模型、前端和表达。
2. **不要跳实操**：每一天的“实操”和“断点”是验收核心，只看文档不算完成。
3. **卡住先看目录**：`00-阅读顺序与使用说明.md`、`01-项目总览.md`、`14-术语表.md` 是地图。
4. **记录问题**：每天用 5 分钟写下“今天最不清楚的三个点”，周末统一查资料或问人。
5. **输出物要可验证**：手画图、SQL 查询结果、断点截图、录音讲解都算输出物。

---

## 学习前置条件

| 条件 | 为什么需要 |
|---|---|
| JDK 17 + Maven / Maven Wrapper | 构建后端 |
| Node LTS + npm | 构建前端 |
| PostgreSQL + pgvector + Redis | 后端最小依赖 |
| IntelliJ IDEA | 读源码、打断点 |
| 浏览器 DevTools | 看 Network、SSE |
| DBeaver / IDEA Database | 查表 |

## 常见问题

**Q：每天必须严格 2～4 小时吗？**
A：是建议值。基础弱可以延长，但不要一次连续学习超过 2 小时不休息。

**Q：可以跳过某些天吗？**
A：不建议。RAG 流程前后依赖强，跳过后面的源码会看不懂。

**Q：要不要实际改代码？**
A：前 13 天以读和跟为主。Day 14 才写改造方案，真正改代码建议在完整理解主流程之后。

**Q：文档和源码对不上怎么办？**
A：以源码为准。把不一致的地方记录下来，作为向项目维护者提问的素材。
