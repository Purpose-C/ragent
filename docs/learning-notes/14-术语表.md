# 术语表

| 术语 | 一句话解释 | 在本项目中的位置 | 相关代码/配置 | 初学者容易误解的点 |
|---|---|---|---|---|
| RAG | 检索增强生成 | 问答 Pipeline | `StreamChatPipeline` | 不只是检索+调用模型 |
| Agent | 能判断并执行步骤的 AI 应用 | 意图、分支、工具 | `IntentResolver` | 不等于聊天机器人 |
| Agentic RAG | 带决策和工具的 RAG | 整条问答链 | `rag/core` | 不一定需要无限自主循环 |
| MCP | 统一发现/调用工具的协议 | MCP client/server | `McpClientAutoConfiguration` | 不是向量数据库 |
| Embedding | 文本到向量 | 入库和查询 | `EmbeddingService` | 不是生成答案 |
| Vector Search | 用向量相似度召回 | pgvector/Milvus | `VectorStoreService` | 不擅长所有精确词匹配 |
| Rerank | 对候选重新排序 | 检索后处理 | `RoutingRerankService` | 不负责首次召回 |
| SSE | 服务端单向流式推送 | 聊天响应 | `SseEmitter`、`useStreamResponse` | 不等同 WebSocket |
| Prompt | 发给模型的指令和上下文 | `resources/prompt` | `RAGPromptService` | 不只是用户问题 |
| Token | 模型处理文本的计量单元 | 历史、上下文、输出 | memory 配置 | 不等于 Java 字符数 |
| LLM | 大语言模型 | 生成、改写、分类 | `LLMService` | 一个模型不适合所有能力 |
| 向量数据库 | 存储并检索向量 | PostgreSQL/Milvus | `rag.vector.type` | pgvector 是 PG 扩展 |
| pgvector | PostgreSQL 向量扩展 | 默认向量方案 | `t_knowledge_vector` | 不是独立数据库进程 |
| 文档切分 | 把长文分成检索块 | 入库节点 | `ChunkerNode` | 越小不一定越好 |
| 文档解析 | 从 PDF 等提取结构化文本 | 入库节点 | `ParserNode`、Tika | 不等于 OCR 全能 |
| 模型路由 | 从候选模型选择调用对象 | `infra-ai` | `ModelSelector` | 不只是随机负载均衡 |
| 熔断 | 暂停调用持续失败的模型 | 模型健康 | `ModelHealthStore` | 不等于永久禁用 |
| 降级 | 主候选失败后使用替代方案 | 模型候选链 | `ModelRoutingExecutor` | 质量可能下降 |
| 重试 | 再次执行同一操作 | SSE/部分客户端 | `useStreamResponse` | 非幂等操作不能盲重试 |
| 幂等 | 重复请求不产生重复副作用 | Controller/AOP | `@IdempotentSubmit` | 不是“请求只执行一次” |
| 会话上下文 | 历史消息和摘要 | memory | `DefaultConversationMemoryService` | 不能无限塞历史 |
| 问题重写 | 把口语/省略问句变清晰 | 检索前 | `MultiQuestionRewriteService` | 可能改变原意，需约束 |
| 意图识别 | 判断问题属于哪类 | 检索前 | `IntentResolver` | 不是关键词 if-else |
| 多路召回 | 多个通道并行找候选 | retrieval | `MultiChannelRetrievalEngine` | 结果要去重融合 |
| 知识库 | 文档和向量的逻辑集合 | knowledge | `t_knowledge_base` | 不是一个文件夹而已 |
| 对象存储 | 保存上传原文件 | RustFS/S3 | `S3FileStorageService` | 不适合直接做语义检索 |
| JSON-RPC | MCP 使用的消息调用格式 | MCP SDK 内部 | `callTool()` | 项目没有手写完整协议栈 |
| Prompt 注入 | 恶意文本诱导模型违背规则 | 文档和用户输入 | 需在 Prompt/工具边界防护 | 检索到的文本也不一定可信 |
| Top-K | 取相关度最高的 K 条 | 检索配置 | `defaultTopK` | K 越大成本和噪声越高 |
| 余弦相似度 | 比较向量方向接近程度 | 向量检索 | `metric-type: COSINE` | 分数不是绝对事实概率 |
| Trace | 记录一次请求各阶段 | 管理后台 | `t_rag_trace_*` | 日志与 Trace 侧重点不同 |
